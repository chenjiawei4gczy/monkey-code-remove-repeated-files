# 安全移动

安全移动是 rmdup 的核心设计理念。工具不直接删除重复文件，而是将其从目录A移动到目录C，保留完整的相对路径结构。这使操作完全可逆。

## 什么是安全移动？

重复文件从目录A被原子性地移动（`Files.move`）到目录C的对应路径下，目录C的目录树结构镜像目录A中被移动文件的位置。

例如，目录A中的 `sub/d.txt` 被判定为重复后，将被移动到 `目录C/sub/d.txt`。

**关键特征**:
- **可逆性**：目录C充当回收站，可以随时将文件移动回目录A
- **结构保留**：目录C保留原始相对路径结构
- **原子操作**：同文件系统内 `Files.move()` 为原子操作

## 实现细节

```java
Path relativePath = baseA.relativize(sourceFile);
Path targetPath = baseC.resolve(relativePath);
Files.createDirectories(targetPath.getParent());
Files.move(sourceFile, targetPath);
```

`MoveResult` 使用密封接口记录每次移动的结果：

```java
public sealed interface MoveResult {
    record Success(Path source, Path destination) implements MoveResult {}
    record Failure(Path source, String reason) implements MoveResult {}
}
```

## 不变量

1. **目录B不受影响**：移动操作仅作用于目录A中的文件
2. **目录A非重复文件保留**：只有被判定为重复的文件被移出
3. **单文件失败不中止**：某个文件移动失败时，记录错误并继续处理其他文件
4. **父目录自动创建**：目标目录C下的父目录按需自动创建
