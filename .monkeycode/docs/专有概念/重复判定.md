# 重复判定

重复判定是 rmdup 识别目录间重复文件的核心规则。一个文件在目录A中被视为重复，当且仅当目录B中存在一个文件与它具有相同的文件名和相同的文件大小。

## 判定依据

**关键特征**:
- 双条件匹配：文件名和文件大小必须同时匹配，缺一不可
- 递归覆盖：判定适用于两个目录的所有子目录层级
- 只读基准：目录B作为参考基准，仅用于建立索引，不会被修改

## 实现细节

`FileKey` 是一个不可变的 Java record：

```java
public record FileKey(String fileName, long fileSize) {}
```

`FileIndex` 将目录B中所有文件按 `FileKey` 建立 `HashMap`，实现对目录A中每个文件的 O(1) 查询。

## 不变量

1. **对称性**：如果文件在之间被移动（从A到C），重复执行工具不会再移出该文件（因为A中的文件已移除）
2. **可判定性**：两个文件只有同时满足文件名相同且文件大小相同时才被判定为重复
3. **基准不变性**：目录B的文件在任何情况下都不会被修改或删除

## 使用场景

| 场景 | 说明 |
|------|------|
| 合并目录 | 将目录A中与目录B重复的文件移出，避免覆盖 |
| 去重清理 | 指定参考目录B，清理目录A中的冗余文件 |
| 构建前准备 | 在部署前清理与已有构建产物重复的文件 |
